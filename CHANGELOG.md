# Changelog

Alle relevanten Änderungen an diesem Projekt werden in diesem Dokument festgehalten.

## [Unreleased]

### Added
- API-Keys werden jetzt automatisch aus `.env`, `config.json` und Environment-Variablen geladen und validiert, bevor sie gespeichert werden.
- Neues Skript `scripts/run_checks.py` bündelt Formatierung, Linting, Tests und Security-Scan.

## So nutzen Sie dieses Changelog

- **Neue Einträge**: Pro Release einen Abschnitt anlegen (z. B. `## [1.0.0] - 2025-01-01`).
- **Kategorien**: Nach Möglichkeit in `Added`, `Changed`, `Fixed`, `Security` unterteilen.
- **Verlinkungen**: Auf PRs oder Issues verweisen (`Fixes #<Issue-Nr>`), damit Nutzer den Kontext nachvollziehen können.
- **Release-Tagging**: Beim Erstellen eines Releases den Abschnitt vom Status `[Unreleased]` in die entsprechende Version kopieren.
